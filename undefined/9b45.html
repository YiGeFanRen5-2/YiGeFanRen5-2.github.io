Java学习笔记（十五）-- 泛型

一、基本介绍

  • 泛型又称参数化类型，是jdk5.0出现的新特性，解决数据类型的安全性问题
  • 在类声明或实例化时只需要指定好需要的具体的类型即可
  • Java泛型可以保证如果程序在编译时没有发出警告，运行时就不会产生ClassCastException异常，使代码更加健壮、简洁

二、泛型的作用

  • 可以在类声明时通过一个标识表示类中某个属性的类型，或者是某个方法的返回值的类型，或者是参数类型

    public class Generic03 {
    public static void main(String[] args) {
        Person<String> tom = new Person<>("tom");
    }
}
//泛型的作用：可以在类声明时通过一个标识标识类中的某个属性的类型，
// 或者是某个方法的返回值的类型，或者是参数类型
class Person<E> {
   //E 表示 S 的数据类型，该数据类型在定义Person对象的时候指定，即在编译期间就确定了E是什么类型 
    E s;
    public Person(E s) {//E 也可以是参数类型
        this.s = s;
    }
    public E f() {//返回类型
        return s;
    }
}

三、泛型的语法

1. 泛型的声明

interface 接口<T> {}// T K V不代表值，而是代表类型
class 类<K,V> {}//任意字母都可以 T 是 Type的缩写

2. 泛型的实例化

//在类后面指定类型参数的值(类型)
List<String> strList = new ArrayList<String>();
Iterator<Customer> iterator = customers.iterator();

四、泛型的使用

1. 注意事项和细节

  • 传入的泛型类型：
(1) interface List<T>{}; public class HashSet<E>{}...//T E只能是引用类型
  • 在指定泛型具体的类型后，可以传入该类型或者其子类型
  • 使用形式：
List<integer> list1 = new ArrayList<integer>();
List<integer> list1 = new ArrayList<>();
  • 若不填写 默认泛型是 E 就是 Object
List list3 = new ArrayList();

五、自定义泛型

1. 自定义泛型类

1.1 基本语法

class 类名<T,R...> {//...表示可以有多个泛型
    //成员
}

1.2 注意细节

  • 普通成员可以使用泛型(属性、方法)
  • 使用泛型的数组不能初始化
  • 静态方法中不能使用类的泛型
  • 泛型类的类型是在创建对象的时候确定的(因为创建对象时，需要指定确定的类型)
  • 如果在创建对象时，没有指定类型，默认为Object

1.3 使用

class Tiger<T,R,M> {
     String name;
    R r;
    M m;
    T t;
    T[] ts;//泛型数组不能初始化，因为数组在new 不能确定T的类型，就无法在内存中开空间
}

2. 自定义泛型接口

2.1 基本语法

interface 接口名 <T,R> {}

2.2 注意细节

  • 接口中，静态成员也不能使用泛型
  • 泛型接口的类型，在继承接口或者实现接口时确定
  • 没有指定类型，默认为Object

2.3 使用

interface IUsb<U,R> {
    //普通方法中可以使用接口泛型
    R get(U u);
    void hi(R r);
    void run(R r1, R r2, U u1, U u2);
    //在jdk8中，可以在接口中使用默认方法
    default R method(U u){
        return null;
    }
}

3. 自定义泛型方法

3.1 基本语法

修饰符 <T,R...> 返回类型 方法名 (参数列表) {
}

3.2 注意细节

  • 泛型方法可以定义在普通方法中，也可以定义在泛型类中
  • 当泛型方法被调用时，类型会确定
  • public void eat(E e){} 修饰符后没有<T,R…> , eat方法不是泛型方法，而是使用了泛型

3.3 使用

class Car{
    public void run(){}//普通方法
    public <T,R> void fly(T t, R r){}//泛型方法
}

六、泛型的继承

  • 泛型不具备继承性

七、泛型的通配符

  • 表示支持任意泛型
  • 表示支持A类以及A类的子类，规定了泛型的上限
  • 表示支持A类及A类的父类，不限于直接父类，规定了泛型的下限
